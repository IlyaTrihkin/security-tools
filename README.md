# 🛡️ linux-audit-engine

Универсальный набор скриптов для автоматизации задач информационной безопасности в Linux.

**Основной инструмент** — `audit.py` — универсальный скрипт аудита безопасности для Linux, который проверяет обновления, устанавливает их (опционально), анализирует логи SSH и генерирует подробный отчёт.

## 1. 🎯 Основной скрипт аудита (`audit.py`)

`audit.py` — это единая точка входа для проведения аудита безопасности на Linux-системах.

**Что он делает:**
1. **Проверяет наличие доступных обновлений** (через `apt` или `yum`).
2. **При наличии обновлений** предлагает их установить (или делает это автоматически с флагом `--yes`).
3. **Анализирует логи SSH** (`/var/log/auth.log`) на предмет неудачных попыток входа (брутфорс).
4. **Выполняет проверку уязвимостей (CVE)** через публичное NVD API для всех или только критических пакетов (с фильтрацией по версиям).
5. **Собирает расширенную системную информацию** (имя хоста, тип ОС, версия ядра, серийный номер, MAC/IP, Machine ID).
6. **Генерирует подробный отчёт** в двух форматах:
   - **TXT** — структурированный.
   - **JSON** — для машинной обработки и интеграции.

## 📦 Установка и использование

### 📋 Требования

- Python 3.6+
- Менеджер пакетов `apt` или `yum`
- Права `sudo` (для установки обновлений и чтения системных файлов)

### Клонирование репозитория

```bash
git clone https://github.com/IlyaTrihkin/security-tools.git
cd security-tools
```

### Установка зависимостей

Рекомендуется использовать виртуальное окружение:
```bash
python3 -m venv venv
source venv/bin/activate
pip install requests packaging tqdm
```

### Запуск

```bash
python audit.py [опции]
```

### ⚙️ Параметры командной строки

| Параметр | Описание |
|----------|----------|
| `--yes, -y` | Автоматически устанавливать все доступные обновления без подтверждения |
| `--no-update` | Пропустить установку обновлений (только проверка) |
| `--cve` | Проверить все установленные пакеты на наличие известных уязвимостей (CVE) через NVD API |
| `--quick` | При проверке CVE проверять только критические пакеты (быстрее) |
| `--refresh-cache` | Принудительно обновить кэш CVE (игнорировать сохранённые данные) |
| `-h, --help` | Показать справку |

### 💻 Примеры использования

#### Базовая проверка (с предложением обновить)

```bash
python3 audit.py
```

#### Автоматическая установка всех обновлений

```bash
python3 audit.py --yes
```

#### Только проверка, без установки

```bash
python3 audit.py --no-update
```

#### Проверка CVE для всех установленных пакетов

```bash
python audit.py --cve
```

#### Быстрая проверка CVE (только критические пакеты)

```bash
python audit.py --cve --quick
```

#### Комбинация: автоматическая установка + быстрая CVE

```bash
python audit.py --yes --cve --quick
```

#### Помощь

```bash
python3 audit.py --help
```

## 🐳 Запуск через Docker

Образ можно собрать и запустить в изолированной среде.

### Сборка образа

```bash
cd docker
docker build -t security-audit -f Dockerfile ..
```

### Запуск контейнера

```bash
# Справка
docker run --rm security-audit

# Проверка CVE (быстрый режим)
docker run --rm security-audit --cve --quick

# Автоматическая установка обновлений (требует --privileged или монтирование)
docker run --rm -it security-audit --yes
```

### Запуск через docker-compose

```bash
cd docker
docker compose up
```
Вы можете изменить команду в docker-compose.yml для нужного набора флагов.


Примечание: внутри контейнера доступ к системным файлам и права sudo ограничены. Для полноценной работы на реальном сервере рекомендуется использовать --privileged и монтировать /var/log и другие системные каталоги.

## 📁 Структура проекта

```bash
linux-audit-engine/
├── audit.py                      # ★ Главный скрипт аудита
├── src/                          # Исходные модули
│   ├── __init__.py
│   ├── system_info.py            # Системная информация (ОС, MAC, IP, Machine ID)
│   ├── updates.py                # Проверка и установка обновлений
│   ├── failed_logins.py          # Анализ логов SSH
│   ├── nvd_analyzer.py           # Проверка CVE через NVD API (с кэшированием, многопоточностью)
│   ├── package_utils.py          # Получение списка установленных пакетов с версиями
│   └── reporter.py               # Генерация отчётов (TXT + JSON)
├── docker/
│   ├── Dockerfile                # Dockerfile для сборки образа
│   └── docker-compose.yml        # Оркестрация
├── reports/                      # Папка для сгенерированных отчётов (создаётся автоматически)
├── .gitignore
├── .dockerignore
└── README.md
```
